🚀 서한F&C 웹사이트 샘플 - 정적 사이트 빌드

⚠️ 현재 서버 환경 문제점

  • PHP 5.3.13p1 - 너무 구버전 (2012년)
  • shell_exec 비활성화 - 서버 명령어 실행 불가
  • Node.js 미설치 - Next.js 실행 불가
  • 복잡한 PHP 스크립트 오류 - 최신 문법 미지원

✅ 권장 해결책: 정적 사이트 빌드

Next.js를 정적 HTML/CSS/JS 파일로 빌드하여 업로드하는 방법입니다.

이 방법은 어떤 서버에서도 작동하며, PHP나 Node.js가 필요하지 않습니다.

🔧 정적 빌드 생성 방법

1단계: 로컬에서 정적 빌드 생성

개발 컴퓨터에서 다음 명령어를 실행하세요:

# next.config.js 수정 (output: 'export' 추가) # 그 다음 빌드 실행 pnpm build # 정적 파일들이 'out' 폴더에 생성됩니다

2단계: 'out' 폴더 내용을 FTP로 업로드

생성된 out 폴더의 모든 내용을 sample 폴더에 업로드하세요.

  • index.html
  • _next/ 폴더
  • images/ 폴더
  • 기타 정적 파일들

3단계: 웹사이트 접속

업로드 완료 후 다음 주소로 접속:

http://www.shfnc.com/sample/

🎯 정적 빌드의 장점

  • ✅ 어떤 서버에서도 작동 - PHP, Node.js 불필요
  • ✅ 빠른 로딩 속도 - 정적 파일 서빙
  • ✅ 안정성 - 서버 오류 없음
  • ✅ 보안 - 서버 사이드 취약점 없음
  • ✅ CDN 호환 - 전 세계 빠른 접속

⚠️ 정적 빌드의 제한사항

  • ❌ 관리자 페이지 기능 제한 - 파일 수정 불가
  • ❌ 인트라넷 로그인 불가 - 서버 사이드 인증 필요
  • ❌ 동적 데이터 업데이트 불가 - 실시간 변경 불가
  • ❌ API 기능 없음 - 서버 API 호출 불가

🔄 대안 방법들

방법 1: 다른 호스팅 사용

  • Vercel - Next.js 전용 호스팅 (무료)
  • Netlify - 정적 사이트 호스팅 (무료)
  • GitHub Pages - 정적 사이트 호스팅 (무료)
  • NHN 클라우드 - Node.js 지원 호스팅

방법 2: 호스팅 업체 문의

현재 호스팅 업체에 다음을 요청:

  • PHP 8.x 버전 업그레이드
  • Node.js 설치
  • shell_exec 함수 활성화
  • SSH 접속 권한

📞 즉시 실행 가능한 방법

지금 당장 할 수 있는 것:

  1. 로컬에서 pnpm build 실행
  2. out 폴더 생성 확인
  3. FileZilla로 out 폴더 내용을 sample에 업로드
  4. http://www.shfnc.com/sample/ 접속 테스트
